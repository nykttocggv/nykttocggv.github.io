Javaで開発したWebアプリケーションで欠かせない"Servlet"。

最近Servletの役割及びメソッドについて学習したので、備忘録として残しておきます。

Servletとは

Servletとは、サーバサイドでJavaプログラムから動的なWebページを生成するための仕組みです。

Javaアプリケーションの場合、実行はJVMによってなされますが、Servletの場合はWebコンテナと呼ばれるものによって実行されます。

Servletの実行の流れ

Servletが実行される時、以下の図のような流れで実行されます。

  1. クライアント(Webブラウザユーザー)がURLにアクセスすると、WebサーバにHTTPリクエストが送信される
  2. Webサーバでrequestを解析する
  3. Webコンテナにrequest情報が渡る
  4. request情報を元にServletが実行
  5. Webサーバにresponseとしてデータが返される
  6. WebサーバからクライアントへHTTPレスポンスが送信され、ユーザはWebページを閲覧できる

というながれになります。

Servletを使うと出来ること

Servletを利用することによるメリットは、Web上でユーザが入力したデータを反映できる という点です。

例えば、コメント機能をページ上に実装することもできます。

  1. ユーザがformに入力する
  2. 入力したデータがrequestと一緒にServletへ渡される
  3. JDBC経由でデータベースにコメント情報が格納される
  4. Servletでデータ処理をしてHTML要素へレンダリングする
  5. Webページ上で投稿コメントが反映される

という流れになるのかな？

つまり、Webページの内容を状況に合わせて変えられる(動的であるということ) 点がServletのメリットです。

使用したソフトウェア及び開発環境

今回利用する開発環境は以下の通りです。

Eclipseで動的プロジェクトの構築

ここからは、実際にServletを動かしてみます。

Eclipseを開き、command + Nで「動的Webプロジェクト」を新規作成。

今回はtestappという名前で作ります。

プロジェクトエクスプローラをみるとtestappの文字が。

次は、そこをアクティブにしてから、再びcommand + Nで「サーブレット」を選択。

今回はパッケージ名を「helloServlet」、クラス名を「HelloServ」にしましょう。

完了を押して新しいサーブレットが開きます。

そこにこちらを記述します。

package helloServlet;

import java.io.IOException;
import java.io.PrintWriter;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

/**
 * Servlet implementation class HelloServlet
 */
@WebServlet("/HelloServlet")
public class HelloServlet extends HttpServlet {

	protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
		response.setContentType("text/html;charset=UTF-8");
		PrintWriter pw = response.getWriter();
		
		pw.println("<html>");
		pw.println("<body>");
		pw.println("<h1>Hello, Servlet!</h1>");
		pw.println("</body>");
		pw.println("</html>");		
	}

}

これで、ソースコードを右クリック→実行→Tomcat 9を選択。

見事、JavaプログラムからHTMLへレンダリングができました。
