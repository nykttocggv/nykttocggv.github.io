システムエンジニアの卵として３ヶ月目に突入しました、chuyaです。

現在、研修中ということでJavaを学習しているのですが、業務ではC#やC++、Kotlinを使うことが多いそう。

って、Javaやってる場合じゃないのでは！？！？！？

と思ったので、より効率よく学習する方法を考えてみることにしました。

同じ悩みで困っている方がもしいれば、この記事読みながら一緒に考えてみませんか。

学習方法を考えてみる

何かを学ぼうと思ったときに、何があるんだろう…?と考えてみると、

  • 参考書で学ぶ
  • 学習サイトで学ぶ
  • 公式ドキュメントを読んで学ぶ
  • プログラミングスクールに入学する
  • etc…

というふうに、思いつく限りはこんな感じですね。

ちょっとそれぞれで考えてみます。

1.参考書で学ぶ

参考書で学習するのが、コストもそれなりに抑えられて、かなり深いところまで学習できる効率的な方法ですよね。

今まで、高校や大学でも、教科書がベースになっていることが多かったはず です。

自分も、研修でJavaを使うときき、真っ先に取り組んだ方法がこれでした。

僕が読んだ参考書は以下の通りです。

これは「ゲームをテーマにJavaを学習する」というのがテーマとなっている本で、例題として出ているプログラムを模写コーディングするだけでゲームを作成 しているかのように学習を進められます。

著者によってはこのように学習者がわかりやすいように工夫が施された本となっていることが多いので、学習する際にはかなり有力となるのではないでしょうか。

学習サイトで学ぶ

例えばこちら。

ドットインストール は、1つの動画が3分前後になっており、尚且つとてもわかりやすく動画がまとめられているので、初心者が学習するにはもってこい なサービスです。

すごくわかりやすいのですが、懸念点は月額料金が発生すること 。

１ヶ月1,000円で短期間集中で学習する分にはかなり有力かもしれません。ただ、

  • 学習ペースが遅い人
  • すでにある程度学習ができていて知識もある人

には 向いていない かもしれません。というのも、

  • 毎月1,000円払って4ヶ月学習するなら、1冊2,000円の参考書を2冊買って4ヶ月ゆっくり好きな時間に学習できる
  • あくまで初心者向けのコンテンツで、物足りない場合が多い

と言った理由があるためです。

場合によって選択すべきですね。

公式ドキュメントを読んで学ぶ

ある程度学習が進んでいる人で、尚且つ英語が読める人 には、かなりおすすめの方法です。

公式ドキュメントとは、例えば、Javaを開発しているOracleから出ているJava Docsのことです。

他にも

言語 公式ドキュメント
C# Microsoft
Python python.org
JavaScript mdn web docs

などがあります。

開発元が公式に出している文書なので、記述内容に誤りがあることが限りなく少ないです。

詳しい仕様や、定義などを知りたい場合にはかなりいいですね。

また、公式ドキュメントにはチュートリアルが含まれている場合も多いです。

これは、Microsoftが出しているC#のチュートリアルです。日本語で書かれていますよね。

ですが、大半は英語で記述されていることが多いので、DeepLなどを用いて翻訳しながら学習する ・ 英語に自信がある と言った人は向いているかもしれません。

情報の正確性は間違いないので、そこが気になる人にとっても最適な方法かな、と思います。

プログラミングスクールを利用する

経済的な余裕があり、尚且つ確実に学習を進めたい時はプログラミングスクールを利用する のも最終手段としてありかもしれません。

例えば、SAMURAI ENGINEERによるSAMURAI TERAKOYA。

こういったプログラミングスクールの特徴としては、

  • 教材が豊富（SAMURAI TERAKOYAの場合、40種類以上あり）
  • Q&A掲示板があるので、わからないことはすぐに相談できる
  • メンター制度の採用

などがあります。

SAMURAI TERAKOYAの場合、ドットインストールなどの動画配信サービスと違う点は、毎月1回無料の60分レッスンを受講できる という点ですね。

Zoomを用いたオンラインレッスンを受けることができるので、普段システムエンジニアとして働いている人でも、わからない点を質問することができます。

無料体験もあるようなので、自分もやってみようかなと思いました。

まとめ

この中でも実際一番効率よく学習できるのは、参考書を購入して学習する方法かなあと思いました。

やっぱり本ってすごいです。

  • 肌身離さず持ち歩いていつでも学習できるようにすること
  • ある程度まとまった時間を確保して学習に取り組むこと

を意識すれば、ある程度学びを進めることができるのではないでしょうか。

しかし、それでもなかなか勉強するのが難しい、という場合は、

  • 経済的な余裕がある → プログラミングスクールの無料体験をやってみる
  • 軽く試してみたい → 動画配信サービスを利用する

と言った方法もありかもしれません。

ぜひ、自分に合った方法を探してみてくださいね。

以上！
