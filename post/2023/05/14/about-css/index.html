【基本情報】クライアントサーバシステムとは

  • 2023-05-14
  • POST
B!

私たちが普段利用しているインターネットやコンピュータシステムでは、常に膨大なデータを処理し続けています。システムがデータを処理するときに、その形態が大まかに２つに分かれてきます。

  • 集中処理 -> データや情報を１箇所にまとめて一括で処理
  • 分散処理 -> 役割を複数のコンピュータごとに決めて効率よく処理すること

これらのうち、今回は分散処理について触れていきます。基本情報でも出題例がそれなりにあるので要チェックです。

分散処理とは何か。

概要

分散処理というのは、複数のコンピュータを同一ネットワークで接続して、データなどの資源を共有することで効率よくデータを処理することができる処理方法です。

具体的には、ロードシェアリングシステム(同じ処理を分担することで負荷を分散)や、後述するクライアントサーバシステムがこれに当たります。

集中処理と比べた時の長所や短所

分散処理のメリットは、

  • 障害発生時のリスクが分散されることで「信頼性」が向上する
  • 複数のコンピュータで運用するので昨日の拡張などが容易である

と言ったメリットがあります。一方、デメリットとしては

  • 複数のコンピュータで構成されているため、障害原因がわかりづらい
  • セキュリティが不十分になりやすい

などが挙げられます。

クライアントサーバシステムとは

分散処理の代表例である「クライアントサーバシステム」とは、３層に分かれた処理層を利用して、役割を分担する手法です。 クライアントとサーバで構成される2層アーキテクチャの間に１層追加することで完成します。

身の回りのクライアントサーバシステムといえば、WebブラウザとWebサーバ が当てはまります。

流れとしては、

  1. ユーザがWebブラウザを操作してリンクにクリック
  2. WebブラウザからWebサーバにページリクエストを送信
  3. リクエストを受け取ったサーバはレンダリングを行い、Webページデータをブラウザにレスポンスする
  4. レスポンスを受け取ったWebブラウザがUI上にWebページを表示
  5. ユーザはリンク先のWebページにアクセスできる

という流れです。

それぞれが役割を分担することで負担分散が実現されています。

こんにちのインターネットでは欠かせない処理体系です。
